# MCP Inspector 테스트 시나리오

## 🧪 필수 테스트 (5개 핵심 Tools)

### 1. search_law
**목적**: 법령 검색 및 약칭 해결 테스트

```json
{
  "query": "관세법"
}
```
**기대 결과**: MST=000013 포함, 법령 목록 출력

```json
{
  "query": "화관법"
}
```
**기대 결과**: "화학물질관리법" 검색 (약칭 자동 해결)

---

### 2. get_law_text
**목적**: 조문 조회 및 JO 코드 자동 변환

```json
{
  "mst": "000013",
  "jo": "제38조"
}
```
**기대 결과**: 관세법 제38조 내용 출력

```json
{
  "mst": "000013",
  "jo": "003800"
}
```
**기대 결과**: 동일한 제38조 내용 (JO 코드로 조회)

---

### 3. parse_jo_code
**목적**: JO 코드 양방향 변환

```json
{
  "joText": "제38조",
  "direction": "to_code"
}
```
**기대 결과**: `{ "input": "제38조", "output": "003800" }`

```json
{
  "joText": "003800",
  "direction": "to_text"
}
```
**기대 결과**: `{ "input": "003800", "output": "제38조" }`

---

### 4. compare_old_new
**목적**: 신구법 대조

```json
{
  "mst": "000013"
}
```
**기대 결과**: 관세법 개정 전후 비교 내용

---

### 5. get_three_tier
**목적**: 3단비교 (법률→시행령→시행규칙)

```json
{
  "mst": "000013",
  "knd": "2"
}
```
**기대 결과**: 관세법의 시행령·시행규칙 위임 조문 계층

---

## 🧪 추가 테스트 (4개 선택 Tools)

### 6. search_admin_rule
**목적**: 행정규칙 검색

```json
{
  "query": "개인정보보호",
  "knd": "2",
  "maxResults": 10
}
```
**기대 결과**: 개인정보보호 관련 예규 목록

---

### 7. get_admin_rule
**목적**: 행정규칙 상세 조회

> ⚠️ search_admin_rule에서 ID를 먼저 획득해야 함

```json
{
  "id": "행정규칙ID"
}
```

---

### 8. get_annexes
**목적**: 별표/서식 조회

```json
{
  "lawName": "관세법",
  "knd": "1"
}
```
**기대 결과**: 관세법 별표 목록

---

### 9. get_ordinance
**목적**: 자치법규 조회

> ⚠️ 자치법규 일련번호가 필요함

```json
{
  "ordinSeq": "자치법규일련번호"
}
```

---

## 🐛 예상 에러 케이스

### E1: 검색 결과 없음
```json
{
  "query": "존재하지않는법령"
}
```
**기대**: "검색 결과가 없습니다. 법령명을 확인해주세요."

### E2: 필수 파라미터 누락
```json
{}
```
**기대**: Zod validation 에러

### E3: 잘못된 JO 코드
```json
{
  "joText": "abc123",
  "direction": "to_code"
}
```
**기대**: "조문 패턴을 인식할 수 없습니다"
